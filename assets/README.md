# Assets

Use this directory for reasonably sized working assets and web-ready files that are appropriate for Git.

Register every material asset in `assets/catalog.yaml`. Keep high-resolution originals, raw media libraries, archives, and restricted assets in an approved object store. Remove unnecessary embedded metadata before publication when appropriate.

Suggested directories may be created as needed:

- `brand/`
- `icons/`
- `photos/`
- `documents/`
- `web/`
